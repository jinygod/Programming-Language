인터페이스(Interface)

(용어)
1. 인터페이스(Interface)는 서로 다른 두 개의 시스템,
   장치 사이에서 정보나 신호를 주고받는 경우의 접점이나 경계면이다.
2. API(Application Programming Interface)
	- Static Library
	- Dynamic Link Library(DLL)
	- Shared Library(SB)
3. IDL(Interface Definition Language)
	- RPC(Remote Procedure Call)
	- CORBA(Common Object Request Broker Architecture)
	- COM(Component Object Model) - OLE
	- DECOM(Distributed Component Object Model)
	- JNI(Java Native Interface)
	- Java Beans
	
4. 	RMI(Java Remote Method Invocation)

5.	JNI(Java Native Interface)
	- C/C++
	
(자바 인터페이스의 특징)
1. 객체의 사용 방법을 정의
2. 객체의 교환성을 높여 주기 위한 역할
3. 객체와 객체사이에 통신하는 접점 역할
4. 함수의 명세(시그니처)만 정의
	- 추상 클래스의 추상 메소드와 같다.
	- 함수의 구현(몸체)가 없다.
5. 구현 클래스를 통해서만 객체화 할 수 있다.
6. 클래스를 규격화(모듈화, 컴포넌트화)하기 위한 목적


(컴포넌트)
- 구현기술 : ActiveX, Java
